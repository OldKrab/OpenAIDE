# Cancel Keeps Task History

When the user stops a Running Task, OpenAIDE will call ACP `session/cancel`, answer pending permission and elicitation waits as canceled where the protocol requires it, stop or release active Host terminals as needed, and keep the Task, Chat history, and already persisted activity. Canceling a turn is not a local-only UI stop. [ADR-0028](0028-native-session-process-isolation.md) refines the execution cleanup: because ACP cancellation cannot guarantee termination of Agent-owned descendants, Stop retires the affected Native Session's isolated process group while preserving the durable session identity and Task history.
